Important FAQ

What is the Document Object Model(DOM)?

DOM represents the structure of an HTML or XML document as a tree of objects, where each object corresponds to a part of the document, like elements, attributes, and text.

How do you select an element from the DOM?

To select an element from the DOM without showing JavaScript code, we would:

1. Identify the element: Each element in a webpage has unique identifiers such as id, class, tag name, etc.
2. Choose a selection method: Use methods like selecting by id, class, or tag. id is unique for an element, while class can apply to multiple elements.
3. Manipulate the selected element: Once selected, we can perform actions like retrieving or modifying its content (e.g., text, HTML, attributes).

What is the event delegation in the context of the DOM, why is it useful?

Event delegation is a technique in the DOM where a single event listener is attached to a parent element, allowing it to manage events from its child elements. Instead of attaching multiple event listeners to each child, the parent captures events that bubble up from them.


Why it's useful:

1. Efficiency: Reduces the number of event listeners, which can improve performance, especially with many elements.
2. Dynamic elements: Handles events on dynamically added child elements, since the parent remains the event handler, even for future child nodes.
3. Simplified code: It centralizes event handling, making the code easier to maintain and less repetitive.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes and styles using the DOM, we can use the following methods:

1. Attributes: getAttribute() is used to read the value of an attribute, while setAttribute() allow to change or add a new attribute.
2. Styles: Styles can also be manipulated by adding, removing, or toggling CSS classes using classList methods like add(), remove(), and toggle().
